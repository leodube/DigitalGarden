### Initial Solution

function reverse(x: number): number {     
    let rev = 0;
    let neg = x <= 0 ? 1 : 0;
    x = Math.abs(x);
    
    while (x) {
        rev = (rev*10) + (x % 10);
        x = (x - (x % 10)) / 10;
    } 
    
    rev = neg ? -rev : rev;
    
    return (rev > 2**31 - 1) || (rev < -(2**31)) ? 0 : rev;
};

###### Intuition:
We can build up the reverse integer one digit at a time. While doing so, we can check beforehand whether or not appending another digit would cause overflow.

##### Algorithm:
Reversing an integer can be done similarly to reversing a string.

We want to repeatedly "pop" the last digit off ofÂ $x$Â and "push" it to the back of theÂ $\text{rev}$. In the end,Â $\text{rev}$Â will be the reverse of theÂ $x$.

1.  IfÂ $temp = \text{rev} \cdot 10 + \text{pop}$Â causes overflow, then it must be thatÂ $\text{rev} \geq \frac{INTMAX}{10}$
2.  IfÂ $\text{rev} > \frac{INTMAX}{10}$, thenÂ $temp = \text{rev} \cdot 10 + \text{pop}$Â is guaranteed to overflow.
3.  IfÂ $\text{rev} == \frac{INTMAX}{10}$, thenÂ $temp = \text{rev} \cdot 10 + \text{pop}$Â will overflow if and only ifÂ $\text{pop} > 7$

    let rev = 0;
    while (x) {
        let pop = x % 10;

    }

};
